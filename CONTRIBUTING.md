# Contributing to Commons OS

Commons OS is a living framework. Contributions from instances strengthen the commons for everyone.

## How to contribute back

1. **Create a branch** in your instance for the contribution
2. **Open a Pull Request** against `commons-engineering/commons-os`
3. **Describe what it is** and why it benefits the commons
4. The Commons Engineering team will review and merge

## What we welcome

| Contribution | Where it goes |
|---|---|
| Collection patterns | `collections/` |
| Blueprint template improvements | `templates/` |
| Workflow improvements | `.github/workflows/` |
| Issue template improvements | `.github/ISSUE_TEMPLATE/` |
| Site theme improvements | `site/` |
| Documentation | `README.md`, `BOOT.md`, inline docs |

## What stays in your instance

Your `.commons/`, `blueprint/`, `inventory/`, `valuestreams/`, `operations/`, `AGENT.md`, and `ALIGN.md` are yours. Don't submit those upstream — they're your sovereign space.

## Conventions

- **Tables over lists** in markdown
- **Agent-neutral language** — no product names in shared files
- One idea per PR. Keep it focused.

## Questions?

Open an issue at [commons-engineering/commons-os](https://github.com/Commons-Engineering/commons-os/issues).
